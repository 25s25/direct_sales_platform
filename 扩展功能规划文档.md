## 一、当前架构梳理

### 1.1 后端现状
- 框架：Spring Boot 3.5 + JDK 17，多模块 Maven 工程
- 业务模块：`ds-common`、`ds-system`、`ds-member`、`ds-product`、`ds-order`、`ds-bonus`、`ds-finance`、`ds-report`、`ds-app`
- 认证授权：Sa-Token 1.40，Redis 共享 Session
- ORM：MyBatis-Plus 3.5.9
- 缓存：Redis 7
- 消息队列：RabbitMQ 3
- 工具库：Hutool 5.8.32
- 配置中心：`ds_sys_config` 表（key-value）

### 1.2 前端现状
- 框架：NuxtJS 3.12 + Vue 3.4 + TypeScript
- UI：Element Plus 2.7
- 状态管理：Pinia
- HTTP：`composables/useApi.ts`（axios 封装）
- 代理：`/api` → `http://localhost:8080`

### 1.3 当前缺口
- 文件上传：缺失，商品图片只能填写 URL
- 在线支付：仅支持余额扣减，无真实第三方支付
- 验证码：无邮箱/短信验证码能力
- 第三方登录：无微信/企业微信登录
- `ds_member` 表无 `email` 字段

## 二、总体设计原则

1. **配置驱动**：所有第三方密钥、开关、回调地址统一存入 `ds_sys_config`，后台可动态修改。
2. **策略抽象**：上传、支付、验证码、第三方登录均定义统一接口，各渠道独立实现。
3. **最小侵入**：以新增模块/类/表为主，不破坏现有会员、订单、奖金结算逻辑。
4. **安全合规**：
   - 验证码限流、文件类型/大小校验
   - 支付回调幂等、签名验证
   - 敏感配置不返回前端
5. **可观测**：所有第三方调用记录操作日志，支付流水可追溯。

**支付宝**

### 4.6 各渠道详细设计

#### 4.6.1 支付宝当面付（ALIPAY_F2F）
- 场景：PC 后台/前台展示二维码
- 流程：
  1. 创建支付 → 调用 `AlipayTradePrecreateRequest`
  2. 返回 `qr_code` 给前端
  3. 前端生成二维码
  4. 用户扫码支付
  5. 支付宝异步回调 `/api/pay/callback/alipay`
- 依赖：`alipay-sdk-java`

#### 4.6.2 支付宝网页支付（ALIPAY_WEB）
- 场景：PC 浏览器跳转
- 细分：
  - `PC`: `AlipayTradePagePayRequest`
  - `WAP`: `AlipayTradeWapPayRequest`
- 流程：
  1. 创建支付 → 返回 HTML form 字符串
  2. 前端 `document.write` 自动提交
  3. 支付宝跳转回 `return_url`
  4. 异步回调通知结果
- 返回：`PaymentResult.formHtml`

#### 4.6.3 微信 Native 支付（WECHAT_NATIVE）
- 场景：PC 端展示微信支付二维码
- 流程：
  1. 调用微信支付 V3 `native` 下单接口
  2. 返回 `code_url`
  3. 前端生成二维码
  4. 用户扫码支付
  5. 微信异步回调 `/api/pay/callback/wechat`
- 依赖：`wechatpay-java`

#### 4.6.4 微信公众号支付（WECHAT_JSAPI）
- 场景：微信内置浏览器 H5
- 前置条件：用户已微信授权获取 `openid`
- 流程：
  1. 调用微信支付 V3 `jsapi` 下单接口，传入 `openid`
  2. 返回 `prepay_id`
  3. 后端组装前端 JS-SDK 调起参数：`appId`、`timeStamp`、`nonceStr`、`package`、`signType`、`paySign`
  4. 前端调用 `WeixinJSBridge.invoke('getBrandWCPayRequest', ...)`
  5. 微信异步回调

#### 4.6.5 微信小程序支付（WECHAT_MINIAPP）
- 场景：微信小程序内
- 前置条件：小程序登录获取 `openid`
- 流程：
  1. 调用微信支付 V3 `jsapi` 下单接口，传入 `openid`
  2. 返回 `prepay_id`
  3. 后端组装 `wx.requestPayment` 所需参数
  4. 前端调起小程序支付
  5. 微信异步回调
- 注意：与公众号支付共用 JSAPI 下单，但 `appid` 不同，签名返回字段也有差异

#### 4.6.6 PayPal
- 场景：国际用户 PC 支付
- 流程：
  1. 创建订单 → 调用 PayPal Orders API v2
  2. 返回 `approval_url`
  3. 前端跳转
  4. 用户确认后 PayPal 回调 `return_url`
  5. 后端捕获订单 `capture`
  6. 更新支付状态

#### 4.6.7 余额支付（WALLET）
- 复用现有 `MemberService.deductWallet`
- 在支付模块中统一封装为 `WalletPayClient`

### 4.9 支付流程时序

```
用户下单 → 订单状态=待付款
   ↓
选择支付方式 → POST /api/pay/create
   ↓
生成 ds_payment_order（状态=待支付）
   ↓
调用对应 PayClient.createPayment
   ↓
返回收银台参数：
  - 支付宝当面付：qr_code
  - 支付宝网页：formHtml
  - 微信Native：code_url
  - 微信JSAPI/小程序：prepay_id + 调起参数
  - PayPal：approval_url
   ↓
用户完成支付
   ↓
第三方异步回调 /api/pay/callback/{channel}
   ↓
验签 → 幂等校验 → 更新 payment_order=成功
   ↓
调用 OrderService.paySuccess(orderNo)
   ↓
更新 ds_order 状态=已付款，记录 pay_time
   ↓
发送 RabbitMQ 消息触发奖金结算
```

## 七、模块五：微信登录

### 7.1 需求
支持三种微信登录场景：
- 微信网页扫码登录（微信开放平台）
- 微信公众号内授权登录（公众号 OAuth2）
- 微信小程序登录（wx.login 获取 code）

### 7.2 数据表
```sql
CREATE TABLE ds_member_social (
    id          BIGINT PRIMARY KEY AUTO_INCREMENT,
    member_id   BIGINT       NOT NULL COMMENT '会员ID',
    social_type VARCHAR(20)  NOT NULL COMMENT 'wechat_web/wechat_mp/wechat_miniapp/workwechat',
    union_id    VARCHAR(100) COMMENT '微信 unionid',
    open_id     VARCHAR(100) NOT NULL COMMENT '平台 openid',
    nickname    VARCHAR(100) COMMENT '昵称',
    avatar      VARCHAR(500) COMMENT '头像',
    raw_data    JSON         COMMENT '原始用户信息',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,

```

### 7.3 核心抽象
```java
public interface SocialAuthClient {
    SocialType type();
    String getAuthUrl(String redirectUri, String state);
    SocialUserInfo getUserInfo(String code);
}
```

### 7.4 配置项
| 配置键 | 说明 |
|---|---|
| `social.wechat.web.enabled` | 网页扫码登录 |
| `social.wechat.web.app-id` | 开放平台 AppID |
| `social.wechat.web.secret` | 开放平台 Secret |
| `social.wechat.mp.enabled` | 公众号登录 |
| `social.wechat.mp.app-id` | 公众号 AppID |
| `social.wechat.mp.secret` | 公众号 Secret |
| `social.wechat.miniapp.enabled` | 小程序登录 |
| `social.wechat.miniapp.app-id` | 小程序 AppID |
| `social.wechat.miniapp.secret` | 小程序 Secret |
| `social.wechat.redirect-uri` | 统一回调地址 |

### 7.5 登录流程

#### 网页扫码
```
1. 前端 GET /api/auth/social/url?type=wechat_web&redirectUri=...
2. 后端返回 https://open.weixin.qq.com/...
3. 用户扫码授权
4. 微信跳转到 redirectUri?code=xxx&state=xxx
5. 前端调用 POST /api/auth/social/login {type:wechat_web, code}
6. 后端用 code 换 access_token → 获取用户信息
7. 已绑定：登录返回 token
8. 未绑定：返回需要绑定手机/邮箱，或自动注册
```

#### 公众号内
```
1. 访问公众号内 H5 页面
2. 后端引导用户授权获取 code
3. 用 code 获取 openid 和用户信息
4. 后续同网页扫码
```

#### 小程序
```
1. 小程序前端调用 wx.login 获取 code
2. 传给后端 POST /api/auth/social/login {type:wechat_miniapp, code}
3. 后端调用 auth.code2Session 获取 openid/session_key
4. 已绑定：登录
5. 未绑定：需前端再传手机号或引导绑定
```

### 7.6 后端 API
| 方法 | 接口 | 说明 |
|---|---|---|
| GET | `/api/auth/social/url` | 获取授权 URL |
| POST | `/api/auth/social/login` | 第三方登录 |
| POST | `/api/auth/social/bind` | 绑定到当前会员 |
| POST | `/api/auth/social/unbind` | 解绑 |

### 7.7 前端页面
- `pages/login.vue`：增加微信登录入口
- `pages/auth/social-callback.vue`：微信回调处理
- 会员中心：账号绑定管理
